# Каскадное перенаправление трафика (iptables)

Простой скрипт для проброса трафика через промежуточный VPS.

```
Клиент → Промежуточный VPS → Удалённый VPS
```

Полезно когда зарубежный сервер заблокирован — клиент подключается к вашему Промежуточный VPS, а тот прозрачно пересылает трафик дальше .

---

## Требования

- Linux (Debian / Ubuntu)
- Права root
- Два сервера: промежуточный (этот) и удалённый (целевой)

---

## Настройка

Откройте `install.sh` и заполните три переменные в начале файла:

```bash
TARGET_IP="XXX.XXX.XXX.XXX"   # IP Удалённого VPS
TARGET_PORT="XXXXX"           # Порт на Удалённом VPS
IN_PORT="XXXXX"               # Порт на этом сервере (Промежуточный VPS) (можно такой же)
PROTO="udp"                   # udp — WireGuard/AmneziaWG, tcp — VLESS/XRay
```

---

## Запуск

```bash
chmod +x install.sh
sudo bash install.sh
```

или если уже зашел как root 

```bash
chmod +x install.sh
./install.sh
```

Скрипт автоматически:
- Включит IP Forwarding
- Настроит правила nftables
- Сохранит правила (переживут перезагрузку)

---

## Настройка клиента

После запуска скрипта в приложении клиента (WireGuard, AmneziaWG, VLESS и т.д.) замените только IP-адрес **Удаленного VPS** на IP этого **Промежуточного VPS**. Порт и остальные настройки не трогайте.
